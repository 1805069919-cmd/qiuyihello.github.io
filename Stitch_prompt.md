# Prompt for Stitch: RIA's Personal Portfolio Prototype

Design a high-fidelity, minimalist personal portfolio website prototype for a user named "RIA". The visual style should draw inspiration from "Terminal / Cyberpunk Dark" aesthetics (like those found on designprompts.dev) combined with an ultra-minimalist layout.

**[Visual Style & Color Palette]**
- **Theme:** Ultra-minimalist, Dark mode, Retro-futuristic.
- **Main Background Color:** Pure Black (#000000) with Neon Green accents.
- **Accent Color:** Bright Neon Green (e.g., #39FF14) for buttons, borders, and active states.
- **Typography Color:** Primary headings and key text MUST use a striking linear gradient transitioning from Dark Green -> Neon Green -> Cyan Blue.

**[Layout & Page Structure]**
1. **Header (Fixed at the top):**
   - Left side: A minimalist LOGO consisting of a single bold uppercase letter.
   - Right side (Fixed position): A clean navigation menu with items: [Home], [About], [Contact], [Portfolio].

2. **Hero Section (Home):**
   - Centered, massive, bold typography reading exactly: "HI HERE IS RIA" (using the specified green-to-blue gradient).
   - Directly below the text: A prominent "Contact Me" button with a Bright Neon Green outline or glow effect.

3. **About Page:**
   - A creative, minimalist layout using typography and grid structures to introduce the user. Keep it spacious with lots of negative space.

4. **Contact Page:**
   - **Avatar Placeholder:** Include a wireframe box, a blank circle, or an abstract geometric shape (STRICTLY NO actual human photos, faces, or realistic avatars).
   - Clean typography listing contact methods: WeChat, GitHub, and Xiaohongshu.

5. **Portfolio Page (Knowledge Notes Organization):**
   - A minimalist bento-box, grid, or clean list layout specifically designed for showcasing "Knowledge Notes" (text-based or structured content). Use abstract wireframe containers instead of actual project screenshots.

**[Interactions & Visual Effects]**
- **Page Transitions:** Smooth fade-in and fade-out effects when clicking navigation items.
- **Scrolling:** Absolutely NO scrolling. The layout should feel like a single-page application where sections replace each other perfectly within the viewport (100vh).
- **Loading/Transition Effect:** Emulate a "Retro CRT TV / Old Television" effect visually during transitions. Incorporate subtle scanlines, static noise textures, or a quick screen-flicker/glitch effect between page changes.

**[Strict Constraints]**
- **NO REAL IMAGES:** Do not generate any real photographs, human faces, or realistic avatars. Use abstract geometry, typography, or empty wireframe boxes only.
- Keep UI elements flat, sharp, and minimal, matching the terminal/coding aesthetic.
